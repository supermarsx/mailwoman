# Internationalization & accessibility foundation

Mailwoman's web UI is localized with [Project Fluent](https://projectfluent.org/)
(`@fluent/bundle` + `@fluent/langneg` + `@fluent/sequence`, Apache-2.0) behind a
thin Solid wrapper in `apps/web/src/i18n/`. This document is the contract every
web feature builds against: the string API, the catalog layout, the RTL/bidi
rules, and the shared accessibility tokens.

## The `t()` API

```ts
import { t, loadCatalog, isolate } from '../i18n';

// 1. Pull your feature area's catalog once, where the area mounts:
onMount(() => void loadCatalog('mail'));

// 2. Use t(id, args?) anywhere — it is REACTIVE inside JSX, so nodes re-render
//    when the catalog finishes loading or the locale changes:
<button>{t('common-ok')}</button>
<h2>{t('mail-unread-count', { count: unread() })}</h2>

// 3. ISOLATE untrusted, user-controlled values (subjects, names, filenames)
//    before interpolating — prevents bidi spoofing (see "RTL & bidi" below):
<span>{t('mail-from', { name: isolate(sender()) })}</span>
```

- `t(id)` and `t(id, args)` return a formatted string. Missing ids return the id
  itself (visible + debuggable) rather than throwing.
- `id`s are **kebab-case and module-prefixed**: `common-ok`, `mail-archive`,
  `calendar-today`. The prefix must match the catalog module.
- Fluent formatting is synchronous once the catalog is loaded; `loadCatalog`
  handles the async part. Reading `t()` in JSX is enough for reactivity.

### `useCatalog` (Suspense)

To defer a screen's first paint until its catalog is present (avoiding an
id→text flash), read the resource inside `<Suspense>`:

```tsx
import { useCatalog } from '../i18n';
function CalendarScreen() {
  const ready = useCatalog('calendar');
  return <Suspense fallback={<Spinner />}>{ready() && <CalendarGrid />}</Suspense>;
}
```

### `useI18n()` (reactive locale/direction)

```ts
import { useI18n } from '../i18n';
const { t, locale, dir, setLocale } = useI18n(); // inside <LocaleProvider>
```

`t` is also available as the module-level import (no provider needed to *read* a
string); `useI18n` adds the reactive `locale()` / `dir()` accessors and
`setLocale(locale)` for a language switcher.

## Adding strings

1. **Add the source string** to the English catalog for your module:
   `apps/web/locales/en/<module>.ftl`. Create the file if your module is new —
   it becomes a Weblate component automatically (see `.weblate/components.yaml`).

   ```ftl
   # apps/web/locales/en/mail.ftl
   mail-archive = Archive
   mail-unread-count = { $count ->
       [one] { $count } unread message
      *[other] { $count } unread messages
   }
   mail-from = From { $name }
   ```

2. **Reference it** with `t('mail-archive')`. Call `loadCatalog('mail')` once
   where the area mounts so the catalog is pulled.

3. **Do NOT** create the non-`en` translations by hand — human translators fill
   those via Weblate. Missing keys fall back to `en` at runtime, so a partial
   catalog is always safe.

### Tests

The global test setup (`src/test/setup.ts`) seeds **all** `en/*.ftl` catalogs and
activates `en` synchronously, so `t(id)` returns the English string in unit
tests with no provider or async. Any `en/<module>.ftl` you add is picked up
automatically (eager glob). Existing literal-text assertions therefore keep
passing as you wrap strings. When asserting against a wrapped string, import the
same accessor:

```ts
import { t } from '../test/i18n';
expect(screen.getByRole('button', { name: t('common-ok') })).toBeInTheDocument();
```

## Catalog layout

```
apps/web/locales/
  en/            # SOURCE — the strings you author
    common.ftl   # cross-cutting (buttons/states) — rides the critical bundle
    mail.ftl     # per-module catalogs, lazily loaded
    calendar.ftl
    …
  de/ fr/ es/ pt-BR/ nl/ it/ pl/ ru/ uk/ zh/ ja/   # translations (Weblate)
```

- **Only `en/common.ftl` is statically imported** (it rides the entry chunk).
  Every other catalog is a lazy `import()` (one Vite chunk per file), pulled by
  `loadCatalog` — so translation weight never inflates the 250 KB login→inbox
  budget. Keep `common.ftl` limited to genuinely cross-cutting strings.

The 12 shipped locales: `en de fr es pt-BR nl it pl ru uk zh ja`.

## RTL & bidi

- Writing direction is resolved per-locale by `resolveDir()` and pushed onto
  `<html dir>` by the shell. None of the 12 current locales are RTL, but the
  plumbing is general: dropping a `locales/ar/` (or `he`, `fa`, `ur`) tree is
  enough to get a mirrored, `dir="rtl"` UI — no code change.
- Author RTL-safe CSS: prefer logical properties (`margin-inline-start`,
  `inset-inline-end`, `text-align: start`) over physical (`margin-left`, …).
- **Isolate untrusted text.** Any user-controlled value interpolated into a
  translated string — email subjects, display names, filenames — must be wrapped
  with `isolate()` from `../i18n`. It surrounds the value with a Unicode
  First-Strong Isolate (FSI…PDI) and strips embedded direction-override
  characters, defeating the RTL-override spoof (`"gnp.exe"` ↔ `"exe.png"`) called
  out in SPEC §24. Fluent's own placeable isolation is intentionally **off** (it
  would inject invisible marks into every interpolation and break copy/paste and
  literal-text assertions), so isolation is surgical and explicit at the call
  site.

## Accessibility token contract

The shared a11y tokens live on the frozen theme contract (`vars.a11y.*`, defined
in `theme/contract.css.ts`, valued in `theme/tokens.ts`). Reference them so a11y
behaviour is themeable and centrally switchable:

| Token | Value | Use |
|---|---|---|
| `vars.a11y.focusRing` | two-stop box-shadow (bg spacer + accent ring) | `:focus-visible { box-shadow: vars.a11y.focusRing }` |
| `vars.a11y.focusRingWidth` | `2px` (→ `3px` under high contrast) | custom outlines |
| `vars.a11y.focusRingColor` | theme accent | custom focus styling |
| `vars.a11y.touchTarget` | `24px` | `min-width`/`min-height` on interactive targets (WCAG 2.2 §2.5.8) |
| `vars.a11y.motionDuration` | `150ms` (→ `0.01ms` under reduced motion) | transitions/animations |
| `vars.a11y.motionDurationSlow` | `240ms` (→ `0.01ms` under reduced motion) | dialog/expand motion |

- **Reduced motion**: authoring transitions against `motionDuration(Slow)` makes
  them collapse automatically when the user prefers reduced motion. The switch is
  driven by both `@media (prefers-reduced-motion: reduce)` and the
  `<html data-reduced-motion>` flag the shell sets (`shell/rootAttributes.ts`), and
  a belt-and-braces global rule neutralizes raw CSS animation on existing chrome.
- **High contrast**: `focusRing`/`focusRingWidth` thicken under
  `@media (prefers-contrast: more)` and `forced-colors: active`. High-contrast
  themes (`hc-light`, `hc-dark`) remain selectable independently.
- **Focus visibility**: always give interactive elements a visible
  `:focus-visible` state using `vars.a11y.focusRing`.
- **Touch targets**: interactive controls must be at least
  `vars.a11y.touchTarget` (24×24 CSS px) in both dimensions.

## Weblate

`weblate.ini` (wlc client) + `.weblate/components.yaml` (component config-as-code)
define the project: Fluent format, `en` source, the 12 gate locales as targets,
component discovery so each `en/<module>.ftl` is its own component. Machine-
translation seeds land in non-`en` catalogs marked `# MT-SEED — NOT REVIEWED`
(added by the i18n CI track) so Weblate flags them for human review.
